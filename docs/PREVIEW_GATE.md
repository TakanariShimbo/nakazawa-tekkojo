# 関係者限定プレビュー・ゲート

公開準備中に、URLを知らない素人が辿り着いても中身を見せないための
**クライアント側のソフトなパスワード目隠し**。`index.html` に組み込み済み。

> ⚠️ これは「本物の認証」ではありません。GitHub Pages は静的サイトでサーバ認証を持てないため、
> 開発者がブラウザの開発者ツールで `#nk-gate` を消す／ソースを読めば中身は見られます。
> 「素人がパスを求められて諦める」レベルの目隠しです（ご要望どおりの強度）。
> 本公開時はこのゲートを削除してください。

## 使い方
- 既定パスワード: **`nakazawa2026`**（友人にだけ口頭/DM等で伝える）
- 正しく入れると `sessionStorage` に記録され、そのタブの間は再入力不要
- **localhost では自動スキップ**（自分の開発時は素通り）。`?lock=1` を付けると強制表示でき、見た目確認に使える

## パスワードの変更
平文ではなく SHA-256 ハッシュで判定しています。新しいパスワードのハッシュを生成して差し替えます。

```bash
node -e "console.log(require('crypto').createHash('sha256').update(process.argv[1]).digest('hex'))" 'あたらしいパスワード'
```

出力されたハッシュを `index.html` 内の以下の行に貼り替える：
```js
var HASH = '（ここに新しいハッシュ）';
```

## 本公開時の解除
`index.html` の `<!-- ▼ 関係者限定プレビュー・ゲート ... ▲ ゲートここまで -->` のブロック
（`<style>` 内の `#nk-gate{...}` 一式も）を削除すれば、誰でも見られる通常公開になります。
